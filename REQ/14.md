## Description

Evaluate and migrate the LLM inference backend from **Ollama** to either **vLLM** or **llama.cpp** for improved performance, reliability, and structured output (guided generation). This evaluation is critical for production deployment after the MVP.

**Why this is needed:**
- Ollama works for MVP with **71% accuracy** on `agresion_particular` classification (from #10 evaluation).
- Guided JSON generation would **force the LLM to output valid JSON** matching `relato.schema.json` exactly — eliminating parse errors and improving schema adherence.
- Ollama does **not support guided JSON** generation.
- vLLM supports guided JSON via `guided_json` with outlines/lm-format-enforcer backends.
- llama.cpp supports guided JSON via **GBNF grammar** files.
- 14B Instruct could be tested (the true `Qwen2.5-14B-Instruct` is available on HuggingFace, but not in Ollama). Note: `qwen2.5-coder:14b` in Ollama performed **worse** (43% accuracy) than 7B (71%) in #10 evaluation.

**This is a Post‑MVP task.** For the hackathon, Ollama remains the production backend.

## Implementation Plan

### Phase 1: vLLM Setup (Post-MVP)

#### 1.1 ROCm 7.13 + vLLM Setup

- [ ] Add AMD ROCm 7.13 repository and install alongside 7.2.0:
  ```bash
  # Add ROCm 7.13 repo
  wget https://repo.radeon.com/amdgpu-install/7.13/ubuntu/jammy/amdgpu-install.deb
  sudo apt install ./amdgpu-install.deb
  # Install ROCm userspace only (don't touch kernel driver)
  sudo amdgpu-install --usecase=rocm --rocmrelease=7.13 --no-dkms
  ```
- [ ] Verify installation: `/opt/rocm-7.13.0/bin/rocminfo`
- [ ] Install PyTorch for gfx1200 + ROCm 7.13:
  ```bash
  uv venv /opt/vllm-venv --python 3.12
  uv pip install --python /opt/vllm-venv/bin/python3 \
    --index-url https://repo.amd.com/rocm/whl/gfx120X-all/ \
    "torch==2.10.0+rocm7.13.0" \
    "torchvision==0.25.0+rocm7.13.0"
  ```
- [ ] Install vLLM via uv (critical: use wheels.vllm.ai to avoid CUDA torch):
  ```bash
  uv pip install --python /opt/vllm-venv/bin/python3 \
    vllm==0.19.1 \
    --extra-index-url https://wheels.vllm.ai/rocm/0.19.1/rocm7130
  ```
- [ ] Set environment variables:
  ```bash
  export FLASH_ATTENTION_TRITON_AMD_ENABLE=TRUE
  export PYTORCH_ROCM_ARCH=gfx1200
  export HSA_OVERRIDE_GFX_VERSION=12.0.0
  ```
- [ ] Verify GPU access:
  ```bash
  /opt/vllm-venv/bin/python3 -c "import torch; print(torch.cuda.device_count(), 'GPUs')"
  ```
- [ ] Test with Qwen2.5-7B-Instruct:
  ```bash
  /opt/vllm-venv/bin/vllm serve Qwen/Qwen2.5-7B-Instruct \
    --host 0.0.0.0 --port 8000 \
    --dtype auto --max-model-len 8192
  ```

#### 1.5.2 GBNF Grammar for relato.schema.json

Create `doc/relato.gbnf`:

```gbnf
# doc/relato.gbnf
# Grammar for SIVeL 3 pre-alert JSON output

root ::= "{" ws "relatos" ws ":" ws "[" ws relato ws "]" ws "}"

relato ::= "{" ws 
  "organizacion_responsable" ws ":" ws string ws "," ws
  "derechos" ws ":" ws string ws "," ws
  "id_relato" ws ":" ws string ws "," ws
  "forma_compartir" ws ":" ws string ws "," ws
  "titulo" ws ":" ws string ws "," ws
  "hechos" ws ":" ws string ws "," ws
  "fecha" ws ":" ws string ws "," ws
  "departamento" ws ":" ws string ws "," ws
  "municipio" ws ":" ws string ws "," ws
  "personas" ws ":" ws "[" ws persona ("," ws persona)* ws "]" ws "," ws
  "grupos" ws ":" ws "[" ws grupo ("," ws grupo)* ws "]" ws "," ws
  "actos" ws ":" ws "[" ws acto ("," ws acto)* ws "]" ws "}"

persona ::= "{" ws "id_persona" ws ":" ws string ws "," ws 
  "nombre" ws ":" ws string ws "," ws
  "apellido" ws ":" ws string ws "," ws
  "sexo" ws ":" ws string ws "}"

grupo ::= "{" ws "id_grupo" ws ":" ws string ws "," ws
  "nombre_grupo" ws ":" ws string ws "}"

acto ::= "{" ws "agresion" ws ":" ws string ws "," ws
  "agresion_particular" ws ":" ws string ws "," ws
  "id_victima_individual" ws ":" ws string ws "," ws
  "id_presunto_grupo_responsable" ws ":" ws string ws "}"

string ::= "\"" ([^"]*) "\""
ws ::= ([ \t\n\r])*
```

## Notes for the Developer

### vLLM Notes
- **ROCm version mismatch:** The host has ROCm 7.2.0 but vLLM needs 7.13 for gfx1200. Docker is the only safe path — upgrading host ROCm would break Ollama.
- **Docker GPU access:** Requires `--device /dev/kfd --device /dev/dri --group-add=video` flags. Test with `docker run --rm --device /dev/kfd --device /dev/dri rocm/dev-ubuntu-22.04 rocminfo` before running vLLM.
- **Model storage:** Mount `$HOME/models` into container to share downloaded models between Ollama and vLLM.
- **VRAM budget:** 16 GB. Q4_K_M 14B uses ~9 GB, leaving ~7 GB for KV cache. vLLM's PagedAttention uses VRAM more efficiently.
- **14B Instruct note:** The true `Qwen2.5-14B-Instruct` is available on HuggingFace but not in Ollama. vLLM would allow testing it. The `qwen2.5-coder:14b` variant in Ollama performed worse (43%) than 7B (71%) in #10.

### llama.cpp Notes
- **Build for ROCm:** Use `-DGGML_HIP=ON -DAMDGPU_TARGETS=gfx1200` for RX 9060 XT support.
- **OpenBSD/adJ:** Use `llamafile` (Cosmopolitan Libc) for native OpenBSD/adJ execution without Docker.
- **GBNF grammar:** Supports full JSON schema validation. Create a grammar file from `relato.schema.json`.
- **Performance:** Optimized for RX 9060 XT (gfx1200) with matrix cores (KHR_coopmat). Expect ~1900 tokens/s on prompt processing.
- **Alternative (if build fails):** Pre-built binaries for OpenBSD/adJ available via `llamafile` project.

Evaluate and migrate the LLM inference backend from **Ollama** to **vLLM** for improved performance, reliability, and structured output (guided generation). vLLM on AMD ROCm is the recommended stack for production inference on RDNA4 GPUs.

**Why this is needed:**
- Ollama with `qwen2.5:7b-instruct-q4_K_M` works for MVP but the 7B model is at its limit: ~40% accuracy on CINEP `agresion_particular` classification.
- The 14B model (`qwen2.5-coder:14b`, 9 GB) is already downloaded and would improve classification, but Ollama's `api/generate` endpoint has no structured output/guided generation.
- vLLM supports **guided JSON generation** (via `guided_json` or outlines/lm-format-enforcer backends), which would force the LLM to output valid JSON matching `relato.schema.json` exactly.
- AMD recommends vLLM as the production inference engine for ROCm.

## Architecture Decision

**RX 9060 XT (gfx1200) requires ROCm ≥7.13 for vLLM.** Two options:

### Option A: Install ROCm 7.13 on host (Recommended)

Install ROCm 7.13 alongside existing 7.2.0 in `/opt/rocm-7.13.0`. AMD supports side‑by‑side ROCm installations.

**Pros:**
- No Docker overhead. Direct GPU access without container permission issues.
- Ollama continues working via existing ROCm 7.2.0 or can use 7.13.
- Simpler development workflow: single environment, no port mapping.
- `amdgpu-install --rocmrelease=7.13` only updates userspace libs. The kernel driver (ROCk 6.16.13, already loaded) is compatible.

**Cons:**
- Requires download of ROCm 7.13 packages (~5 GB).
- Small risk of `amdgpu-install` updating kernel driver (mitigated by pinning or using `--usecase=rocm` only).

### Option B: Docker container

Use official image `rocm/vllm:rocm7.13.0_gfx120X_...`. Isolates ROCm 7.13 without touching host.

**Pros:** Zero risk to host ROCm installation.
**Cons:** Docker GPU permissions can be problematic on Ubuntu (observed in practice). Overhead of container management.

## Implementation Plan

### Phase 1: ROCm 7.13 + vLLM Setup (Post-MVP)

- [ ] Add AMD ROCm 7.13 repository and install alongside 7.2.0:
  ```bash
  # Add ROCm 7.13 repo
  wget https://repo.radeon.com/amdgpu-install/7.13/ubuntu/jammy/amdgpu-install.deb
  sudo apt install ./amdgpu-install.deb
  # Install ROCm userspace only (don't touch kernel driver)
  sudo amdgpu-install --usecase=rocm --rocmrelease=7.13 --no-dkms
  ```
- [ ] Verify installation: `/opt/rocm-7.13.0/bin/rocminfo`
- [ ] Install PyTorch for gfx1200 + ROCm 7.13:
  ```bash
  uv venv /opt/vllm-venv --python 3.12
  uv pip install --python /opt/vllm-venv/bin/python3 \
    --index-url https://repo.amd.com/rocm/whl/gfx120X-all/ \
    "torch==2.10.0+rocm7.13.0" \
    "torchvision==0.25.0+rocm7.13.0"
  ```
- [ ] Install vLLM via uv (critical: use wheels.vllm.ai to avoid CUDA torch):
  ```bash
  uv pip install --python /opt/vllm-venv/bin/python3 \
    vllm==0.19.1 \
    --extra-index-url https://wheels.vllm.ai/rocm/0.19.1/rocm7130
  ```
- [ ] Set environment variables:
  ```bash
  export FLASH_ATTENTION_TRITON_AMD_ENABLE=TRUE
  export PYTORCH_ROCM_ARCH=gfx1200
  export HSA_OVERRIDE_GFX_VERSION=12.0.0
  ```
- [ ] Verify GPU access:
  ```bash
  /opt/vllm-venv/bin/python3 -c "import torch; print(torch.cuda.device_count(), 'GPUs')"
  ```
- [ ] Test with Qwen2.5-7B-Instruct:
  ```bash
  /opt/vllm-venv/bin/vllm serve Qwen/Qwen2.5-7B-Instruct \
    --host 0.0.0.0 --port 8000 \
  ```

## Ollama vs vLLM Comparison

| Feature | Ollama | vLLM (Docker) |
|---------|--------|---------------|
| **Setup complexity** | ✅ Trivial (`ollama pull`) | ⚠️ Docker + ROCm 7.13 image |
| **ROCm version** | ✅ Works with 7.2.0 (host) | ❌ Requires 7.13 (containerized) |
| **Guided JSON** | ❌ Not supported | ✅ `guided_json` parameter |
| **Batch inference** | ❌ Single request | ✅ Continuous batching |
| **VRAM efficiency** | ⚠️ Good | ✅ Better (PagedAttention) |
| **API** | Custom (`/api/generate`) | ✅ OpenAI-compatible (`/v1/chat/completions`) |
| **Model hot-swap** | ✅ `ollama pull` | ⚠️ Restart server |
| **MVP readiness** | ✅ Working now | ⏳ Needs Docker setup |

## Notes

- **ROCm version mismatch:** The host has ROCm 7.2.0 but vLLM needs 7.13 for gfx1200. Docker is the only safe path — upgrading host ROCm would break Ollama.
- **Docker GPU access:** Requires `--device /dev/kfd --device /dev/dri --group-add=video` flags.
- **Model storage:** Mount `$HOME/models` into container to share downloaded models between Ollama and vLLM.
- **VRAM budget:** 16 GB. Q4_K_M 14B uses ~9 GB, leaving ~7 GB for KV cache. vLLM's PagedAttention uses VRAM more efficiently.
- **Alternative (if Docker not viable):** Build vLLM from source against host ROCm 7.2.0 with `PYTORCH_ROCM_ARCH=gfx1200`. This is complex and not officially tested.

Integrate a lightweight LLM (Qwen2.5-7B-Instruct) as the AI engine for the
`sivel3agent` MVP running on **Ollama** with **ROCm 7.2.0** on an
**AMD Radeon RX 9060 XT** (gfx1200, 32 CUs). For the hackathon, the LLM
performs **only one core task**:

1. **Extract pre‑alerts** from raw news articles (structured JSON conforming to
   `relato.schema.json`)

**Excluded from MVP (post‑MVP):**
- Embeddings generation (we use simple `event_hash` for deduplication)
- Pre‑alert completion (`NeedsMoreInfo` state – out of MVP scope)
- Semantic similarity (handled by pgvector, not needed for MVP)

**Why Qwen2.5-7B-Instruct?**
- **Instruct vs Coder:** Instruct is a general-purpose assistant optimized for
  understanding instructions and extracting structured data (JSON).
  Coder (`qwen2.5-coder:14b`) is specialized for code generation and less
  suitable for this task.
- **Context window:** 131,072 tokens (~100K words) — more than enough for any
  news article without truncation.
- 7B uses ~5 GB VRAM (fits easily in RX 9060 XT with 9 GB available).
- Faster inference than 14B (critical for demo).
- Sufficient for structured JSON extraction from news articles.

---

## Key Design Decisions (MVP)

| Decision | Rationale |
|----------|-----------|
| **Qwen2.5-7B-Instruct (4‑bit)** | Lightweight, fast, good enough for
extraction. Available models: `qwen2.5:7b-instruct-q4_K_M` |
| **Ollama + ROCm 7.2.0** | Local inference on AMD RX 9060 XT (gfx1200).
`qwen2.5-coder:14b` already downloaded but Instruct preferred for JSON task. |
| **No embeddings** | Use `event_hash` (keccak256 of JSON) for duplicate
detection |
| **No completion** | `NeedsMoreInfo` state not needed for MVP |
| **Low temperature (0.2)** | Ensure deterministic output for same input |
| **Full `clean_text` no truncation** | 131K context window handles any article |
| **Single script** | `scripts/extract-and-save.ts` – read from `source`,
extract, store |
| **Ollama URL configurable** | `LLM_URL` from `.env`, not hardcoded |

### 2. Pre‑alert Extraction (MVP only)

Create `lib/extractPreAlert.ts`:

```typescript
// lib/extractPreAlert.ts
import { keccak256, toBytes } from 'viem';

const LLM_URL = process.env.LLM_URL || 'http://localhost:11434/api/generate';
const LLM_MODEL = process.env.LLM_MODEL || 'qwen2.5:7b-instruct-q4_K_M';

interface News {
  title: string;
  date: string;
  text: string;           // clean_text from source table
  sourceUrl: string;
  sourceMedium: string;
}

interface PreAlertResult {
  json: object;           // Conforms to doc/relato.schema.json
  eventHash: string;      // keccak256 of JSON.stringify(json)
}

export async function extractPreAlert(
  news: News,
  maxRetries = 2
): Promise<PreAlertResult> {
  const prompt = buildExtractionPrompt(news);

  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      const jsonText = await callLLM(prompt);
      // Strip markdown fences if present
      const cleaned = jsonText
        .replace(/^```json\s*/i, '')
        .replace(/^```\s*/i, '')
        .replace(/```\s*$/, '')
        .trim();
      const preAlertJson = JSON.parse(cleaned);
      const eventHash = keccak256(toBytes(JSON.stringify(preAlertJson)));
      return { json: preAlertJson, eventHash };
    } catch (err) {
      if (attempt === maxRetries) throw err;
      console.warn(`LLM JSON parse failed (attempt ${attempt + 1}), retrying…`);
    }
  }
  throw new Error('extractPreAlert: max retries exceeded');
}

function buildExtractionPrompt(news: News): string {
  return `
Eres un experto en documentación de violencia política en Colombia y Palestina.
Extrae la siguiente información de esta noticia en formato JSON.
Sigue el esquema de \`doc/relato.schema.json\`.
NO inventes información. Si un campo no aparece, omítelo.

Título: ${news.title}
Fecha: ${news.date}
Fuente: ${news.sourceMedium} (${news.sourceUrl})
Texto: ${news.text}

Devuelve SOLO el JSON, sin explicaciones ni markdown.
  `;
}

async function callLLM(prompt: string): Promise<string> {
  const response = await fetch(LLM_URL, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      model: LLM_MODEL,
      prompt: prompt,
      stream: false,
      options: { temperature: 0.2, top_p: 0.9 }
    })
  });

  if (!response.ok) {
    throw new Error(`Ollama API error: ${response.status} ${response.statusText}`);
  }

  const data = await response.json();
  return data.response;
}
```

### 3. Script: Extract and Save

Create `scripts/extract-and-save.ts`:

```typescript
// scripts/extract-and-save.ts
import { newKyselyPostgresql } from '../.config/kysely.config.js';
import { extractPreAlert } from '../lib/extractPreAlert';

async function extractAndSave() {
  const db = newKyselyPostgresql();

  // Get unprocessed sources (where clean_text is not null and no pre_alert
  // linked yet). Qwen2.5-7B-Instruct has 131K token context, so we pass the
  // full clean_text without truncation.
  const sources = await db
    .selectFrom('source')
    .select(['id', 'url', 'title', 'published_at', 'clean_text', 'medium'])
    .leftJoin('pre_alert_source', 'source.id', 'pre_alert_source.source_id')
    .where('pre_alert_source.pre_alert_id', 'is', null)
    .where('clean_text', 'is not', null)
    .limit(10)  // Process in batches
    .execute();

  for (const source of sources) {
    console.log(`Processing: ${source.title}`);
    
    // Extract pre-alert
    const { json, eventHash } = await extractPreAlert({
      title: source.title,
      date: source.published_at?.toISOString().split('T')[0] || '',
      text: source.clean_text,
      sourceUrl: source.url,
      sourceMedium: source.medium || 'Unknown'
    });
    
    // Check duplicate by event_hash (simple, no embeddings needed for MVP)
    const dupCheck = await db
      .selectFrom('pre_alert')
      .select('id')
      .where('event_hash', '=', eventHash)
      .executeTakeFirst();

    if (dupCheck) {
      console.log(`  → Duplicate detected (event_hash), skipping.`);
      continue;
    }

    // Save pre_alert
    const preAlert = await db
      .insertInto('pre_alert')
      .values({
        event_hash: eventHash,
        json_data: json,
        status: 'pending'
      })
      .returning('id')
      .executeTakeFirst();
    
    // Link source to pre_alert
    await db
      .insertInto('pre_alert_source')
      .values({
        pre_alert_id: preAlert!.id,
        source_id: source.id
      })
      .execute();
    
    console.log(`  → Saved pre_alert ID: ${preAlert!.id} (${eventHash})`);
  }
}

extractAndSave()
  .then(() => process.exit(0))
  .catch((error) => {
    console.error('Fatal error:', error);
    process.exit(1);
  });
```

## Description

Set up the foundational Next.js project structure for `sivel3agent`, the AI agent that will autonomously detect socio‑political violence events and generate pre‑alerts for the SIVeL 3 ecosystem.

This is the foundation for all other agent features (scraper, LLM integration, duplicate detection, and sync with sivel.xyz).

## Notes for the developer

- **Do not modify `bin/m` commands.** All agent‑specific logic will go into `scripts/` and be executed via `npx tsx scripts/...`.
- **Database:** The agent runs on its own machine, so it has its own PostgreSQL instance. No need to isolate from sivel3 – they are physically separate.
- **Wallet security:** The agent wallet (`sivel3agent`) will be used later to sign on‑chain transactions. Never commit the private key. Store it only in `~/.m/wallets/` (managed by `bin/m wallet:generate`). All wallet operations go through `bin/m wallet:*`.
- **Git:** The AI agent must not execute write git commands. Only the human maintainer commits and pushes.

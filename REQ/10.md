Define and implement a systematic evaluation framework to compare candidate LLMs for the core extraction task: converting raw news articles into structured JSON pre‑alerts conforming to `doc/relato.schema.json`.

The goal is to select the best model for the MVP based on a multi‑dimensional assessment that goes beyond simple accuracy:

- **Precision**: Accuracy of extracted fields (titulo, hechos, fecha, departamento, municipio, etc.) using tolerance‑based matching for textual fields.
- **Path Efficiency**: How efficiently the model produces the output (output length, number of retries) – a proxy for certainty and reliability.
- **Speed**: Inference time per article.
- **Resource usage**: VRAM consumption and memory footprint.
- **Context handling**: Ability to process long news articles (> 4000 tokens) without truncation.
- **JSON validity**: Output is valid JSON and conforms to the schema.

**Decision:** The final model will be chosen based on a weighted score of these metrics, with precision being the highest priority, but with consideration of path efficiency to avoid models that "guess" with verbose, error‑prone outputs.

---

## Golden Dataset

The evaluation dataset will be built from **already documented cases** from the Banco de Datos del CINEP/PPP and the Network of Data Banks. For each case, we will identify the original source (news article, communiqué, report) that served as input for documentation.

**Why this approach?**
- It aligns with the real methodology used by the Data Bank, which relies on sources such as press, social organizations, and official complaints.
- The JSON structure (`relato.schema.json`) is based on the CINEP conceptual framework for classifying human rights violations, IHL breaches, and political‑social violence.
- The model will learn to extract information from the same types of inputs it will process in production.

### Sources for the Dataset

We will use sources that are already documented in CINEP/PPP reports, such as:

| Source Type | Examples | Region |
|-------------|----------|--------|
| **National press** | El Tiempo, El Espectador, RCN, Caracol | National |
| **Local media** | Conexión Putumayo, MiPutumayo Noticias | Putumayo |
| **Social organizations** | Red de DDHH Putumayo, Comisión Intereclesial | Putumayo, National |
| **International reports** | ONU‑DH, HRW | National |
| **Official documents** | Defensoría del Pueblo (Early Warnings) | National |

### Selection Criteria

- **Time period:** Cases documented between 2020–2026 (covering recent years, with a focus on 2025‑2026 for the MVP).
- **Geographic diversity:** Putumayo, Cauca, Antioquia, Nariño, Bogotá.
- **Violation types:** DDHH (extrajudicial executions, torture, disappearances), IHL (forced displacement, attacks on civilians), VPS (threats, political persecution, sexual violence).
- **Source diversity:** Mix of national press, local media, communiqués, and reports.

### Annotation Process

For each selected source, we will manually create the expected JSON following `relato.schema.json`. The schema includes fields defined by the CINEP conceptual framework:

- `titulo`: Short summary
- `hechos`: Description (who + what + to whom + circumstances)
- `fecha`: Date of event
- `departamento`, `municipio`: Geographic location
- `personas`: Victims or affected individuals
- `grupos`: Responsible groups (armed groups, state forces, etc.)
- `actos`: Type of aggression (e.g., extrajudicial execution, forced displacement)
- `contextos`: Contextual factors (e.g., persecution of social organizations, peace processes)

**Example from the Data Bank methodology:** A news article about the murder of a social leader in Putumayo would be annotated with `titulo`, `hechos`, `fecha`, `departamento: Putumayo`, `municipio: Puerto Asís`, `personas: [nombre del líder]`, `grupos: [paramilitares, Ejército]`, `actos: [VIDA, ejecución extrajudicial]`, `contextos: [persecución a organizaciones sociales]`.

### Dataset Size

We will create a **golden dataset** of **30–50 articles** manually annotated. This size is sufficient for a comparative evaluation of candidate models. For fine‑tuning, we will expand this dataset to 500–1000 examples (using the same methodology, but with semi‑automated annotation).

## Metrics

### 1. Precision (Primary, 40% weight)

We evaluate **tolerance‑based matching** for textual fields, not exact string match. This accounts for paraphrasing while preserving semantic correctness.

| Field | Weight | Evaluation method |
|-------|--------|-------------------|
| `titulo` | 15% | Exact match (case‑insensitive, trimmed) |
| `hechos` | 25% | **Tolerance‑based:** Levenshtein similarity > 0.85 |
| `fecha` | 10% | Exact match (ISO format) |
| `departamento` | 10% | Exact match (against official list) |
| `municipio` | 10% | Exact match (against official list) |
| `personas` | 15% | Jaccard similarity > 0.7 |
| `grupos` | 10% | Jaccard similarity > 0.7 |
| `actos` | 5% | Jaccard similarity > 0.7 |

**Total precision score:** Weighted average (0–100%).

### 2. Path Efficiency (15% weight)

This metric evaluates the model's path to the final output – a proxy for certainty and reliability.

| Sub‑metric | Description | Scoring |
|------------|-------------|---------|
| **Output length** | Total tokens generated (including thinking / reasoning). | Shorter is better. Penalise outputs > 2× expected JSON length. |
| **Number of retries** | How many malformed JSON attempts before success. | 0 retries = 100%, 1 retry = 50%, ≥2 retries = 0%. |
| **Hallucination markers** | Phrases like "I think", "probably", "maybe", "could be". | Deduct 10% per occurrence (max penalty 30%). |

### 3. Speed (15% weight)

- **Inference time** per article (end‑to‑end, seconds)
- **Tokens per second** (generation speed)

### 4. Resource usage (15% weight)

- **VRAM usage** (peak, GB)
- **Model size** (disk, GB)

### 5. Context handling (10% weight)

- **Long article test:** Process a 5000‑word article and check if output is complete (no truncation).
- **Graceful degradation:** Does performance drop significantly with longer texts (> 2000 words)?

### 6. JSON Validity (5% weight)

- **Valid JSON** (parseable)
- **Valid against `relato.schema.json`** (using `ajv`)

### Overall score formula

```
Total Score = (Precision * 0.40) + (Path Efficiency * 0.15) + (Speed * 0.15) + (Resources * 0.15) + (Context * 0.10) + (JSON * 0.05)
```

Where each metric is normalised to a 0–100 scale.

// lib/modelRunner.ts
const OLLAMA_URL = process.env.OLLAMA_URL || 'http://localhost:11434/api/generate';

export async function runOllamaModel(
  model: string,
  prompt: string,
  options = { temperature: 0.0, top_p: 0.9 }
): Promise<{ response: string; outputLength: number }> {
  const start = Date.now();
  const response = await fetch(OLLAMA_URL, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      model,
      prompt,
      stream: false,
      options,
    }),
  });

  if (!response.ok) {
    throw new Error(`Ollama error: ${response.status} ${response.statusText}`);
  }

  const data = await response.json();
  const outputLength = data.response?.length || 0;
  const elapsed = (Date.now() - start) / 1000;
  return { response: data.response, outputLength };
}
```

### 2. Evaluation script

Create `scripts/evaluate-models.ts`:

```typescript
// scripts/evaluate-models.ts
import fs from 'fs';
import path from 'path';
import { runOllamaModel } from '../lib/modelRunner';
import { extractPreAlert } from '../lib/extractPreAlert';
import Ajv from 'ajv';
import schema from '../doc/relato.schema.json';
import { distance } from 'fastest-levenshtein';

const MODELS = [
  'qwen2.5:0.5b-instruct-q4_K_M',
  'qwen2.5:1.5b-instruct-q4_K_M',
  'qwen2.5:7b-instruct-q4_K_M',
  'qwen2.5:14b-instruct-q4_K_M',
  'qwen3.5:4b-instruct-q4_K_M',
  'deepseek-r1:7b-q4_K_M',
  'deepseek-coder-v2-lite:latest',
  'llama3.2:3b-instruct-q4_K_M',
  'mistral:7b-instruct-v0.3-q4_K_M',
  'gemma2:9b-instruct-q4_K_M',
];

const FIELD_WEIGHTS = {
  titulo: 0.15,
  hechos: 0.25,
  fecha: 0.10,
  departamento: 0.10,
  municipio: 0.10,
  personas: 0.15,
  grupos: 0.10,
  actos: 0.05,
};

const HALLUCINATION_MARKERS = ['I think', 'probably', 'maybe', 'perhaps', 'could be', 'likely', 'it seems'];

interface Result {
  model: string;
  precision: number;
  pathEfficiency: number;
  speed: number;
  vram: number;
  modelSize: number;
  contextScore: number;
  validJson: number;
  validSchema: number;
}

function getVRAMUsage(model: string): number {
  const vramMap: Record<string, number> = {
    'qwen2.5:0.5b-instruct-q4_K_M': 0.5,
    'qwen2.5:1.5b-instruct-q4_K_M': 1.5,
    'qwen2.5:7b-instruct-q4_K_M': 5.0,
    'qwen2.5:14b-instruct-q4_K_M': 9.0,
    'qwen3.5:4b-instruct-q4_K_M': 3.0,
    'deepseek-r1:7b-q4_K_M': 5.0,
    'deepseek-coder-v2-lite:latest': 4.5,
    'llama3.2:3b-instruct-q4_K_M': 2.0,
    'mistral:7b-instruct-v0.3-q4_K_M': 5.0,
    'gemma2:9b-instruct-q4_K_M': 6.0,
  };
  return vramMap[model] || 4.0;
}

function normalizeText(text: string): string {
  return text.toLowerCase().trim().replace(/\s+/g, ' ');
}

function evaluateField(actual: any, expected: any): { score: number; detail: string } {
  if (actual === undefined || actual === null) {
    return { score: 0, detail: 'missing' };
  }

  if (typeof actual === 'string' && typeof expected === 'string') {
    const normActual = normalizeText(actual);
    const normExpected = normalizeText(expected);
    
    // Exact match after normalisation
    if (normActual === normExpected) {
      return { score: 1, detail: 'exact' };
    }
    
    // Tolerance-based match using Levenshtein
    const maxLen = Math.max(normActual.length, normExpected.length);
    if (maxLen === 0) return { score: 0, detail: 'empty' };
    const levDistance = distance(normActual, normExpected);
    const similarity = 1 - (levDistance / maxLen);
    
    if (similarity >= 0.85) {
      return { score: similarity, detail: `fuzzy: ${similarity.toFixed(2)}` };
    }
    return { score: 0, detail: `low-similarity: ${similarity.toFixed(2)}` };
  }

  if (Array.isArray(actual) && Array.isArray(expected)) {
    const setA = new Set(actual.map(JSON.stringify));
    const setB = new Set(expected.map(JSON.stringify));
    const intersection = new Set([...setA].filter(x => setB.has(x)));
    const score = intersection.size / Math.max(setA.size, setB.size, 1);
    return { score, detail: `jaccard: ${score.toFixed(2)}` };
  }

  // Fallback for other types
  return { score: actual === expected ? 1 : 0, detail: 'strict' };
}

function calculatePathEfficiency(
  outputLength: number,
  retries: number,
  responseText: string
): { score: number; detail: string } {
  const expectedLength = 800;
  let score = 1.0;
  
  if (outputLength > expectedLength * 2) {
    const penalty = Math.min(0.5, (outputLength - expectedLength * 2) / (expectedLength * 2));
    score -= penalty * 0.5;
  }
  
  if (retries === 0) score -= 0;
  else if (retries === 1) score -= 0.3;
  else score -= 0.5;
  
  const markersFound = HALLUCINATION_MARKERS.filter(m => responseText.includes(m));
  if (markersFound.length > 0) {
    const penalty = Math.min(0.3, markersFound.length * 0.1);
    score -= penalty;
  }
  
  return {
    score: Math.max(0, Math.min(1, score)) * 100,
    detail: `length=${outputLength}, retries=${retries}, markers=${markersFound.length}`
  };
}

async function evaluateModel(model: string, dataset: any[]): Promise<Result> {
  let total = 0;
  let fieldScores: Record<string, { sum: number; details: string[] }> = {};
  let totalTime = 0;
  let totalOutputLength = 0;
  let totalRetries = 0;
  let validJsonCount = 0;
  let validSchemaCount = 0;

  const ajv = new Ajv();
  const validate = ajv.compile(schema);

  for (const item of dataset) {
    const start = Date.now();
    let retries = 0;
    let json: object | null = null;
    let responseText = '';
    
    while (retries <= 2) {
      try {
        const result = await runOllamaModel(model, buildExtractionPrompt(item.news));
        responseText = result.response;
        const cleaned = responseText
          .replace(/^```json\s*/i, '')
          .replace(/^```\s*/i, '')
          .replace(/```\s*$/, '')
          .trim();
        json = JSON.parse(cleaned);
        break;
      } catch {
        retries++;
        if (retries > 2) {
          json = null;
          break;
        }
      }
    }

    const elapsed = (Date.now() - start) / 1000;
    totalTime += elapsed;
    totalOutputLength += responseText.length;
    totalRetries += retries;

    const isValidJson = json !== null;
    if (isValidJson) validJsonCount++;

    const isValidSchema = json && validate(json);
    if (isValidSchema) validSchemaCount++;

    if (isValidJson) {
      for (const [field] of Object.entries(FIELD_WEIGHTS)) {
        const result = evaluateField((json as any)?.[field], item.expected[field]);
        if (!fieldScores[field]) fieldScores[field] = { sum: 0, details: [] };
        fieldScores[field].sum += result.score;
        fieldScores[field].details.push(result.detail);
      }
    }
    total++;
  }

  let precisionSum = 0;
  for (const [field, weight] of Object.entries(FIELD_WEIGHTS)) {
    const avg = fieldScores[field]?.sum / total || 0;
    precisionSum += avg * weight;
  }

  const avgOutputLength = totalOutputLength / total;
  const avgRetries = totalRetries / total;
  const sampleResponse = dataset.length > 0 ? await runOllamaModel(model, buildExtractionPrompt(dataset[0].news)) : { response: '' };
  const pathEff = calculatePathEfficiency(avgOutputLength, avgRetries, sampleResponse.response);

  const contextMap: Record<string, number> = {
    'qwen2.5:0.5b-instruct-q4_K_M': 32768,
    'qwen2.5:1.5b-instruct-q4_K_M': 32768,
    'qwen2.5:7b-instruct-q4_K_M': 131072,
    'qwen2.5:14b-instruct-q4_K_M': 131072,
    'qwen3.5:4b-instruct-q4_K_M': 262144,
    'deepseek-r1:7b-q4_K_M': 128000,
    'deepseek-coder-v2-lite:latest': 128000,
    'llama3.2:3b-instruct-q4_K_M': 128000,
    'mistral:7b-instruct-v0.3-q4_K_M': 32768,
    'gemma2:9b-instruct-q4_K_M': 8192,
  };
  const contextLen = contextMap[model] || 0;
  const contextScore = Math.min(100, (contextLen / 100000) * 100);

  return {
    model,
    precision: precisionSum * 100,
    pathEfficiency: pathEff.score,
    speed: totalTime / total,
    vram: getVRAMUsage(model),
    modelSize: contextLen / 1000000,
    contextScore,
    validJson: (validJsonCount / total) * 100,
    validSchema: (validSchemaCount / total) * 100,
  };
}

function buildExtractionPrompt(news: any): string {
  return `
Eres un experto en documentación de violencia política en Colombia y Palestina.
Extrae la siguiente información de esta noticia en formato JSON.
Sigue el esquema de \`doc/relato.schema.json\`.
NO inventes información. Si un campo no aparece, omítelo.

Título: ${news.title}
Fecha: ${news.date}
Fuente: ${news.sourceMedium} (${news.sourceUrl})
Texto: ${news.text}

Devuelve SOLO el JSON, sin explicaciones ni markdown.
  `;
}

async function main() {
  const dataset = JSON.parse(fs.readFileSync('test/golden-dataset.json', 'utf-8'));
  const results: Result[] = [];

  for (const model of MODELS) {
    console.log(`\n🔍 Evaluating ${model}...`);
    try {
      const result = await evaluateModel(model, dataset);
      results.push(result);
      console.log(`  Precision: ${result.precision.toFixed(2)}%`);
      console.log(`  Path Efficiency: ${result.pathEfficiency.toFixed(2)}%`);
      console.log(`  Speed: ${result.speed.toFixed(2)}s/article`);
      console.log(`  VRAM: ${result.vram} GB`);
    } catch (error) {
      console.error(`  ❌ Failed: ${error}`);
    }
  }

  console.log('\n📊 SUMMARY');
  console.log('='.repeat(80));
  console.table(results.map(r => ({
    Model: r.model,
    Precision: r.precision.toFixed(2) + '%',
    'Path Eff.': r.pathEfficiency.toFixed(2) + '%',
    Speed: r.speed.toFixed(2) + 's',
    VRAM: r.vram + ' GB',
    Context: r.contextScore.toFixed(0) + '%',
    ValidJSON: r.validJson.toFixed(1) + '%',
    ValidSchema: r.validSchema.toFixed(1) + '%'
  })));

  const maxSpeed = Math.max(...results.map(x => x.speed));
  const maxVram = Math.max(...results.map(x => x.vram));

  const scored = results.map(r => {
    const normPrecision = r.precision / 100;
    const normPathEff = r.pathEfficiency / 100;
    const normSpeed = maxSpeed > 0 ? 1 - (r.speed / maxSpeed) : 1;
    const normVram = maxVram > 0 ? 1 - (r.vram / maxVram) : 1;
    const normContext = r.contextScore / 100;
    const normJSON = (r.validJson + r.validSchema) / 200;
    
    const overall = (
      normPrecision * 0.40 +
      normPathEff * 0.15 +
      normSpeed * 0.15 +
      normVram * 0.15 +
      normContext * 0.10 +
      normJSON * 0.05
    ) * 100;
    
    return { ...r, overall };
  });

  console.log('\n🏆 OVERALL SCORE (weighted)');
  console.log('='.repeat(80));
  const sorted = scored.sort((a, b) => b.overall - a.overall);
  sorted.forEach((r, i) => {
    console.log(`${i+1}. ${r.model}: ${r.overall.toFixed(2)}%`);
  });

  const best = sorted[0];
  console.log(`\n✅ Recommended: ${best.model} (${best.overall.toFixed(2)}%)`);

  fs.writeFileSync('evaluation-results.json', JSON.stringify(sorted, null, 2));
  console.log('\n📄 Results saved to evaluation-results.json');
}

main().catch(console.error);

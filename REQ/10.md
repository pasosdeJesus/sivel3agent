# REQ/10 — Model Evaluation Framework

Define and implement a systematic evaluation framework to compare candidate LLMs for converting raw news articles into structured JSON pre‑alerts following the **Noche y Niebla** methodology from the Banco de Datos del CINEP.

## Metrics (Noche y Niebla alignment)

| Metric | Weight | What it measures |
|--------|--------|------------------|
| **Classification accuracy** | 40% | `agresion_particular` code number matches ground truth (normalized) |
| **Geographic precision** | 25% | `departamento` (15%) + `municipio` (10%) exact match |
| **Actor identification** | 10% | `grupos` name present and matches expected |
| **Context match** | 5% | `contextos` overlap with expected |
| **JSON validity** | 10% | Parseable JSON |
| **Speed** | 10% | Normalized: faster = better |

## Results (June 2026)

### SKILL-based prompt (final)

| # | Modelo | Score | Agresión | Depto | Municipio | Velocidad |
|---|--------|-------|----------|-------|-----------|-----------|
| 1 | **7B Instruct** | **71.3%** | 5/7 (71%) | 6/7 | 4/7 | 7.1s |
| 2 | 14B Instruct/Coder | 60.7% | 3/7 (43%) | 7/7 | 5/7 | 15.5s |
| 3 | 14B Coder | 60.3% | 3/7 (43%) | 6/7 | 5/7 | 14.6s |
| 4 | 1.5B Instruct | 53.2% | 1/7 (14%) | 6/7 | 5/7 | 3.4s |
| 5 | 3B Instruct | 44.2% | 0/7 (0%) | 6/7 | 5/7 | 4.7s |

**JSON validity: 100% across all models.** Zero retries needed.

### Key Findings

1. **7B beats 14B in classification (71% vs 43%).** Counter‑intuitive. 14B over‑thinks and picks complex DIHC codes (D703, D903) when a simple B40 would match. 7B is more conservative and reliable.

2. **14B-instruct and 14B-coder are the same GGUF.** Verified via `ollama show`: identical architecture (`qwen2`), parameters (14.8B), context (32K), and quantization (Q4_K_M). The true `Qwen2.5-14B-Instruct` is not available in Ollama's registry — only the Coder variant. This invalidates the 14B comparison.

3. **3B worse than 1.5B (0% vs 14% agresión).** The 3B model may be poorly calibrated for structured extraction or overfitted to chat tasks.

4. **Prompt SKILL does not improve over simple prompt.** We tested two prompts: (a) simple list of codes, (b) condensed SKILL with classification rules (estatal→A, no estatal político→B, conflicto→C/D). Results were identical. The bottleneck is model capacity, not prompt engineering.

5. **Geolocation is easy.** Even 1.5B gets 6/7 departamentos. Extracting named locations from text is a solved problem for all model sizes.

6. **Actor identification is the hardest task.** Best model only got 1/7. Groups like "Comandos de la Frontera" are mentioned in the source text but models fail to extract them reliably.

7. **Speed scales linearly with size.** 1.5B is 4.5× faster than 14B. For batch processing hundreds of articles, 7B is the sweet spot.

## Additional Metrics

| Metric | Weight | Description |
|--------|--------|-------------|
| **Reasoning** | 10% | Ability to infer relationships between acts and contexts (e.g., "threat" + "social leader" → "persecution of organization") |
| **Consistency** | 5% | Does the model produce the same output for the same input across 3 runs? |
| **Hallucination** | 5% | Does the model invent information (e.g., names of victims not present in text)? |

**Updated weights (sum = 100%):**

| Metric | Weight |
|--------|--------|
| Classification (agresión) | 30% |
| Geographic | 20% |
| Actors | 10% |
| Context | 5% |
| JSON validity | 10% |
| Speed | 10% |
| Reasoning | 10% |
| Consistency | 5% |

**Note:** `phi4:14b` has only 16K context, which may truncate long articles (>4000 tokens). Results for that model should be flagged if truncation occurs.

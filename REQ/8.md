## Description

Configure the `sivel3agent` to run automatically at regular intervals. For the MVP, the agent runs three scripts in sequence: **scrape → detect → send**.

**Current state (June 20, 2026):** Scripts run manually. Pipeline verified end‑to‑end (36 pre‑alerts sent). Cron not yet configured.

**Simplifications for MVP:**
- Simple cron job using system crontab (no Kubernetes, no complex schedulers)
- Manual setup on the GPU server
- Logging to file (no centralized logging)
- No health checks or retries (will be added post‑MVP)

**Post‑MVP:** Add Docker/Kubernetes scheduling, health checks, retry logic, and monitoring.

---

## Pipeline

The cron job executes three scripts in sequence:

1. **`scrape-news.ts`** — RSS scraper (idempotent, 15 active feeds). `make scrape`
2. **`detect-cases.ts`** — Case detection (LLM‑heavy, ~30s/article). `make detect`
3. **`generate-and-send.ts`** — Sign + send to sivel.xyz (reads `pre_alert WHERE status='pending'`). `make send`

**Frequency:** `scrape-news.ts` can run every 6 hours (fast). `detect-cases.ts` can run daily or after scrape (slow, uses LLM).

## Implementation

### 1. Shell Script Wrapper

Create `scripts/run-agent.sh`:

```bash
#!/bin/bash
# scripts/run-agent.sh
# Wrapper for the sivel3agent cron job. Replace PROJECT_DIR with the
# actual installation path (e.g., /opt/sivel3agent in production).

set -e

PROJECT_DIR="${SIVEL3AGENT_HOME:-/opt/sivel3agent}/apps/nextjs"
LOG_DIR="${SIVEL3AGENT_LOG_DIR:-/var/log/sivel3agent}"
LOG_FILE="$LOG_DIR/agent.log"
LOCK_FILE="/var/lock/sivel3agent.lock"

mkdir -p "$LOG_DIR"

if [ -f "$LOCK_FILE" ]; then
    echo "$(date): Another instance is already running, exiting." >> "$LOG_FILE"
    exit 0
fi
trap "rm -f $LOCK_FILE" EXIT
touch "$LOCK_FILE"

cd "$PROJECT_DIR" || exit 1

echo "=========================================" >> "$LOG_FILE"
echo "$(date): Starting sivel3agent pipeline" >> "$LOG_FILE"

# 1. Scrape news (idempotent, fast)
echo "$(date): [1/3] Scraping RSS feeds..." >> "$LOG_FILE"
DOTENV_CONFIG_PATH=../.env node_modules/.bin/tsx scripts/scrape-news.ts >> "$LOG_FILE" 2>&1

# 2. Detect cases (LLM-heavy, slow)
echo "$(date): [2/3] Detecting cases..." >> "$LOG_FILE"
DOTENV_CONFIG_PATH=../.env BATCH_SIZE=20 node_modules/.bin/tsx scripts/detect-cases.ts >> "$LOG_FILE" 2>&1

# 3. Send to sivel.xyz
echo "$(date): [3/3] Sending to sivel.xyz..." >> "$LOG_FILE"
node_modules/.bin/tsx scripts/generate-and-send.ts >> "$LOG_FILE" 2>&1

echo "$(date): Pipeline completed" >> "$LOG_FILE"
echo "=========================================" >> "$LOG_FILE"
```

**Make it executable:**
```bash
chmod +x scripts/run-agent.sh
```

### 2. Test the Script Manually

```bash
cd /path/to/sivel3agent/apps/nextjs
./scripts/run-agent.sh
tail -f /var/log/sivel3agent/agent.log
```

### 3. Configure Cron

Edit the crontab for the user that runs the agent (e.g., `root` or a dedicated `sivel3agent` user):

```bash
crontab -e
```

Add the following line to run every 6 hours:

```cron
# sivel3agent - run every 6 hours at minute 0
0 */6 * * * /path/to/sivel3agent/apps/nextjs/scripts/run-agent.sh
```

**Example schedule (every 6 hours):**

| Time (UTC) | Local (Colombia, UTC-5) |
|------------|--------------------------|
| 00:00 | 19:00 (previous day) |
| 06:00 | 01:00 |
| 12:00 | 07:00 |
| 18:00 | 13:00 |

**Alternative schedules for demo (more frequent during testing):**

```cron
# Every hour (for testing)
0 * * * * /path/to/run-agent.sh

# Every 30 minutes (for testing)
*/30 * * * * /path/to/run-agent.sh
```

### 4. Verify Cron is Running

```bash
# List current crontab
crontab -l

# Check cron logs (Ubuntu/Debian)
grep CRON /var/log/syslog | tail -20

# Check agent log
tail -f /var/log/sivel3agent/agent.log
```

## Environment Considerations

| Issue | Workaround |
|-------|------------|
| **`pnpm` not found in cron** | Use absolute path: `/usr/local/bin/pnpm` or source profile |
| **Environment variables not loaded** | Source `.env` file manually in the wrapper script |
| **Node.js version mismatch** | Use `#!/bin/bash -l` to load login shell environment |

**Robust wrapper with environment detection:**

```bash
#!/bin/bash -l
# scripts/run-agent.sh (with environment detection)

# Load Node.js/pnpm if using nvm
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"

# Use absolute paths
PNPM=$(which pnpm || echo "/usr/local/bin/pnpm")
PROJECT_DIR="/path/to/sivel3agent/apps/nextjs"
LOG_FILE="/var/log/sivel3agent/agent.log"

cd "$PROJECT_DIR" || exit 1

# Load .env
set -a
source .env
set +a

$PNPM tsx scripts/generate-and-send.ts >> "$LOG_FILE" 2>&1
```

## Log Rotation (Optional but Recommended)

Create a logrotate configuration to prevent the log file from growing indefinitely.

File: `/etc/logrotate.d/sivel3agent`

```
/var/log/sivel3agent/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 0640 root root
}
```

## Description

Implement duplicate detection for pre‑alerts generated by the AI agent. The goal is to prevent the same event from being published multiple times.

**For MVP:** Use **simple exact matching** based on `event_hash` (keccak256 of the JSON). This works for detecting duplicates within the agent's own generated pre‑alerts.

**For Noche y Niebla integration (#3):** Use **semantic similarity** with pgvector to detect if a generated pre‑alert matches an existing case in the Noche y Niebla database (60,000+ cases).

**Post‑MVP:** When processing historical news (pre‑2026) or handling ambiguous duplicates, we will expand semantic similarity to cover other sources.

---

## Simplified Approach (MVP)

### 1. Hash-based duplicate detection (agent‑side)

When the agent generates a new pre‑alert:

1. Compute `event_hash = keccak256(JSON.stringify(preAlert))`.
2. Query the `pre_alert` table for existing records with the same `event_hash`.
3. If a record exists → **discard** the new pre‑alert (do not publish).
4. If not → proceed to publish.

**Why this works for MVP:**
- The same news article (source content) will produce the same JSON (deterministic LLM output with low temperature).
- The 2026 date range has no existing cases, so only identical pre‑alerts generated multiple times will be caught.

### 2. Source-based duplicate detection (optional)

Before generating a pre‑alert, the agent can check if the **same source URL** has already been used:

```sql
SELECT 1 FROM source WHERE url = :url OR content_hash = :content_hash
```

If the source already exists, skip processing (unless the agent is explicitly updating a `NeedsMoreInfo` pre‑alert – out of MVP scope).

```

### 2. Case detector deduplication check

Modify `lib/case-detector/index.ts` to check against Noche y Niebla:

```typescript
// lib/case-detector/index.ts
export async function detectCase(article: Article) {
  // 1. Fast filter (keywords)
  // 2. LLM extraction (JSON)
  // 3. Check against Noche y Niebla (embeddings)
  const isDuplicate = await checkNocheYNieblaDuplicate(article.clean_text);
  
  if (isDuplicate) {
    return {
      isCase: false,
      reason: 'Duplicate of Noche y Niebla case',
      category: 'duplicate',
    };
  }
  
  // 4. Continue with case detection...
}

async function checkNocheYNieblaDuplicate(text: string): Promise<boolean> {
  // Generate embedding for the article text
  const embedding = await generateEmbedding(text);
  
  // Search for similar cases in source table (is_case = true, case_reason = 'noche_y_niebla')
  const result = await sql`
    SELECT id, 1 - (embedding <=> ${embedding}::vector) as similarity
    FROM source
    WHERE is_case = true AND case_reason = 'noche_y_niebla'
    ORDER BY similarity DESC
    LIMIT 1
  `.execute(db);
  
  if (result.rows.length === 0) return false;
  
  const similarity = result.rows[0].similarity;
  
  // Thresholds
  if (similarity > 0.85) {
    return true; // Duplicate
  }
  
  if (similarity > 0.70) {
    // Possible duplicate → mark for documenter review
    return false; // Don't block, but flag
  }
  
  return false;
}
```

### 3. Embedding model selection

| Model | Size | Use case |
|-------|------|----------|
| **`BAAI/bge-m3`** | ~2 GB | Multilingual, 1024 dims. Best quality. |
| **`paraphrase-multilingual-MiniLM-L12-v2`** | ~500 MB | 384 dims. Faster, sufficient for MVP. |

**For MVP:** Use `paraphrase-multilingual-MiniLM-L12-v2` (runs on CPU, no VRAM).

### 4. Pre‑alert → Enrichment (not duplicate)

When similarity is 0.70–0.85, the pre‑alert is **not blocked**. Instead:

1. Create the pre‑alert as normal
2. Link it to the existing Noche y Niebla case
3. Mark it as `enrichment_pending` in the database
4. Documenter can review and enrich the existing case

```sql
ALTER TABLE pre_alert ADD COLUMN similar_case_id INTEGER;
ALTER TABLE pre_alert ADD COLUMN similarity_score DECIMAL(5,4);
ALTER TABLE pre_alert ADD COLUMN enrichment_status VARCHAR(20) DEFAULT 'pending';
```

## Implementation

### 1. Helper functions

Create `lib/duplicate-check.ts`:

```typescript
// lib/duplicate-check.ts
import { keccak256, toBytes } from 'viem';
import { db } from './db';
import { generateEmbedding } from './embeddings';

export async function isDuplicatePreAlert(preAlertJson: object): Promise<boolean> {
  const jsonString = JSON.stringify(preAlertJson);
  const eventHash = keccak256(toBytes(jsonString));
  
  const existing = await db
    .selectFrom('pre_alert')
    .select('id')
    .where('event_hash', '=', eventHash)
    .executeTakeFirst();
  
  return !!existing;
}

export async function isDuplicateSource(url: string, contentHash: string): Promise<boolean> {
  const existing = await db
    .selectFrom('source')
    .select('id')
    .where('url', '=', url)
    .orWhere('content_hash', '=', contentHash)
    .executeTakeFirst();
  
  return !!existing;
}

export async function checkNocheYNieblaDuplicate(
  text: string,
  threshold: number = 0.85
): Promise<{ isDuplicate: boolean; similarCaseId?: number; similarity?: number }> {
  const embedding = await generateEmbedding(text);
  
  const result = await sql`
    SELECT id, 1 - (embedding <=> ${embedding}::vector) as similarity
    FROM source
    WHERE is_case = true AND case_reason = 'noche_y_niebla'
    ORDER BY similarity DESC
    LIMIT 1
  `.execute(db);
  
  if (result.rows.length === 0) {
    return { isDuplicate: false };
  }
  
  const similarity = result.rows[0].similarity;
  
  return {
    isDuplicate: similarity > threshold,
    similarCaseId: result.rows[0].id,
    similarity: similarity,
  };
}
```

## Post‑MVP: Semantic Similarity Expansion

When we need to detect similar (not identical) events beyond Noche y Niebla:

### 1. pgvector extension

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

### 2. Add embedding column to pre_alert

```sql
ALTER TABLE pre_alert ADD COLUMN embedding VECTOR(384);
```

### 3. Similarity query across all sources

```sql
SELECT id, 1 - (embedding <=> :embedding) as similarity
FROM pre_alert
WHERE 1 - (embedding <=> :embedding) > 0.85
ORDER BY similarity DESC
LIMIT 5;
```

### 4. Configuration

| Threshold | Action |
|-----------|--------|
| `> 0.95` | Discard (duplicate) |
| `0.85 – 0.95` | Mark `NeedsMoreInfo` (possible duplicate, require documenter review) |
| `0.70 – 0.85` | Flag as enrichment (link to existing case) |
| `< 0.70` | Publish as new pre‑alert |

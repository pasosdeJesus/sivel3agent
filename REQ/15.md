Implement bidirectional converters between the Banco de Datos del CINEP XML format
(`relatos-099.dtd`) and the JSON Schema (`relato.schema.json`). This enables:

1. **Importing** existing documented cases (XML → JSON) for the golden evaluation dataset (#10)
2. **Exporting** AI-generated pre‑alerts (JSON → XML) for integration with CINEP/PPP systems
3. **Roundtrip validation** to ensure fidelity of the conversion

## Implementation

### 1. XML → JSON Converter

**File:** `scripts/convert-xrlat.ts`

**Usage:**
```bash
npx tsx scripts/convert-xrlat.ts entrada.xml > salida.json
```

**Mapping rules:**
- `<relato>` → JSON object with all child elements
- `<persona>`, `<grupo>`, `<victima>` → arrays of objects
- `<acto>` → array with `agresion`, `agresion_particular`, `id_victima_individual`, `id_grupo_victima`, `id_presunto_grupo_responsable`, `id_presunto_responsable_individual`
- `<observaciones tipo="…">` → array of `{ tipo, valor }` objects
- Groups with `id_grupo` 4-25 or 35 → `grupos` (presuntos responsables)
- Groups with custom ids (≥1000) or descriptive names → `grupos_victimizados`
- `<contexto>` via `observaciones tipo="contexto"` → `contextos` array
- Misc observaciones (region, intervalo, tsitio, etc.) → `observaciones_misc` object

**Tested with:** 11 real cases from Putumayo, first semester 2025 (`/tmp/casos-put-2025-I.xrlat`)

### 2. JSON → XML Converter

**File:** `scripts/convert-json-to-xrlat.ts`

**Usage:**
```bash
npx tsx scripts/convert-json-to-xrlat.ts entrada.json [salida.xml]
```

**Mapping rules (inverse of above):**
- JSON `relatos[]` → `<relatos><relato>...</relato></relatos>`
- `personas[]` → `<persona>` elements with `<!-- Victimas personas individuales -->` comment
- `grupos[]` → `<grupo>` elements under `<!-- Presuntos responsables -->`
- `grupos_victimizados[]` → `<grupo>` elements under `<!-- Grupos victimizados -->`
- `victimas[]` → `<victima>` elements
- `actos[]` → `<acto>` elements
- `contextos[]` → `<observaciones tipo="contexto">` elements
- `observaciones_misc` → `<observaciones tipo="…">` elements under `<!-- Otros -->`
- HTML special chars escaped: `&`, `<`, `>`, `"`

**Roundtrip verified:** Original XML → JSON → XML → JSON produces identical field values for all 11 test cases.

## Golden Dataset Pipeline (for #10)

The converters enable this workflow:

```
CINEP/PPP XML export ──▶ convert-xrlat.ts ──▶ JSON
                                                   │
                                          Add fuente URLs manually
                                                   │
                                                   ▼
                                     golden-dataset.json (#10)
```

When you provide source article URLs for each case, we can enrich the JSON with `fuentes` arrays linking to the original news articles. This creates the annotated dataset needed for model evaluation.

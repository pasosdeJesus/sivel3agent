# R-#2 Source table with content + pre_alerts table (MVP)


Create the database tables required for the `sivel3agent` to store pre‑alerts and their **full source evidence**. For the MVP, we **do not** implement a full case cache or embeddings. However, we **must** preserve the original content (raw and cleaned) to provide verifiable evidence for each pre‑alert.

**Scope for MVP:** The agent processes news from **January–June 2026**, a period for which there are no existing cases in `sivel2`.

**Post‑MVP:** When we implement historical case sync, we will add `case_cache` with embeddings.

---

## Tables

### 1. `sources`

Stores original news sources with full content.

```sql
CREATE TABLE sources (
    id SERIAL PRIMARY KEY,
    url VARCHAR(500) UNIQUE NOT NULL,
    medium VARCHAR(100),                    -- e.g., "Periodico Putumayo", "ReliefWeb"
    title VARCHAR(500),
    published_at TIMESTAMP,                 -- fecha de publicación original
    fetched_at TIMESTAMP DEFAULT NOW(),
    content_hash VARCHAR(66) UNIQUE,        -- SHA256 of clean_text
    raw_content TEXT,                       -- HTML or raw JSON (original evidence)
    clean_text TEXT,                        -- Texto limpio (sin HTML) para el LLM
    metadata JSONB                          -- metadata adicional
);

CREATE INDEX idx_sources_url ON sources(url);
CREATE INDEX idx_sources_content_hash ON sources(content_hash);
CREATE INDEX idx_sources_published_at ON sources(published_at);
```

### 2. `pre_alerts`

Stores pre‑alerts generated by the AI agent.

```sql
CREATE TABLE pre_alerts (
    id SERIAL PRIMARY KEY,
    event_hash VARCHAR(66) UNIQUE NOT NULL,  -- keccak256 of the JSON
    json_data JSONB NOT NULL,                -- full pre‑alert JSON
    status VARCHAR(20) DEFAULT 'pending',    -- pending, reserved, converted, expired
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_pre_alerts_event_hash ON pre_alerts(event_hash);
CREATE INDEX idx_pre_alerts_status ON pre_alerts(status);
```

### 3. `pre_alert_sources` (junction table)

```sql
CREATE TABLE pre_alert_sources (
    pre_alert_id INTEGER REFERENCES pre_alerts(id) ON DELETE CASCADE,
    source_id INTEGER REFERENCES sources(id) ON DELETE CASCADE,
    PRIMARY KEY (pre_alert_id, source_id)
);
```

---

## Flujo de trabajo

1. **Scraper** obtiene una noticia:
   - Guarda `url`, `title`, `published_at`, `raw_content`.
   - Limpia HTML → `clean_text`.
   - Calcula `content_hash = sha256(clean_text)`.
   - Si ya existe por `url` o `content_hash`, omite la inserción (evita duplicados).

2. **LLM** extrae la pre-alerta:
   - Lee `clean_text` de la fuente.
   - Genera el JSON de pre-alerta.
   - Inserta en `pre_alerts`.
   - Relaciona con `source_id` en `pre_alert_sources`.

3. **Deduplicación**:
   - Por `content_hash` (mismo texto → misma fuente).
   - Por `event_hash` (mismo JSON → misma pre-alerta).
